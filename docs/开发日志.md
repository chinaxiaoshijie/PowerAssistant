## 2026-03-04 修复：交付健康度 SQL 查询兼容性问题

**问题**: PostgreSQL 不支持 `json_contains()` 函数

运行交付健康度指标计算时报错：
```
ERROR: function json_contains(json, character varying) does not exist
```

**根本原因**: `func.json_contains()` 是 MySQL 语法，PostgreSQL 使用不同的 JSON 操作符。

**修复文件**: `src/services/metrics/delivery_health.py`

```python
# 修复前（错误 - MySQL 语法）
.where(
    func.json_contains(FeishuTask.labels, "bug").is_not(None)
    | func.json_contains(FeishuTask.labels, "issue").is_not(None)
)

# 修复后（正确 - PostgreSQL 原生操作符）
.where(
    (FeishuTask.labels.op("?")("bug"))
    | (FeishuTask.labels.op("?")("issue"))
)
```

同时补充了缺失的导入：
```python
from sqlalchemy import func, or_, select
```

**PostgreSQL JSON 操作符参考**:

| 操作符 | 说明 |
|--------|------|
| `?` | 检查 JSON 对象中是否存在指定 key，或数组中是否包含指定值 |
| `@>` | 包含操作符（JSONB），左侧是否包含右侧所有元素 |
| `->` | 提取 JSON 对象字段（返回 JSON） |
| `->>` | 提取 JSON 对象字段（返回文本） |

**经验教训**: 不同数据库的 JSON 函数差异大，使用 SQLAlchemy 的 `op()` 方法可调用数据库原生操作符，避免使用 `func.*` 调用可能不兼容的函数名。
